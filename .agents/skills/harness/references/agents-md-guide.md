# AGENTS.md Guide

`AGENTS.md` is always-loaded guidance. Keep it durable and small.

## Include

- What the project is and which surfaces matter.
- Repo-wide constraints that apply to most changes.
- Exact verification commands when they are non-obvious.
- Pointers to deeper skills, docs, and harness specs.

## Leave Out

- Generated directory tours.
- Conditional workflow detail that belongs in a skill or team spec.
- Temporary model-specific recovery logic.
- Long copies of style guides already enforced by tooling.

## Shape

Use short sections such as:

- `What`
- `Why`
- `How`

This repository also keeps detailed coding rules in `AGENTS.md`, so new harness details should usually be added as pointers rather than copied procedures.
